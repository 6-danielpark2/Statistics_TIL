```
✅ 학습 목표 :
```

### 11.1. 결측값 처리

결측값(Missing Value) -> 데이터 수집 또는 입력 과정에서 누락된 값으로, 모델의 신뢰도와 분석 정확도를 떨어뜨림

1. 완전 무작위 결측 (MCAR)
    → 결측이 전혀 규칙 없이 무작위로 발생한 경우.

2. 무작위 결측 (MAR)
    → 다른 변수의 특성에 의해 결측이 일정한 패턴으로 발생하는 경우.


3. 비무작위 결측 (NMAR)
    → 결측이 변수 자체의 특성과 관련된 경우.

결측값 처리 방법
---

1. 표본 제거법 (Listwise Deletion)
    → 결측치가 포함된 행이나 열 자체를 제거하는 가장 단순한 방법.
    → 일반적으로 결측 비율이 10% 미만일 때 사용 가능.

2. 단순 대치법 (Simple Imputation)
    → 결측값을 평균, 중앙값, 최빈값 등으로 채워 넣음.
    → 계산이 간단하지만, 분산이 줄어들고 표준오차가 과소 추정되는 단점이 있다.

3. 보간법 (Interpolation)
    → 시계열 데이터에 자주 사용. 이전 값(pad), 다음 값(bfill), 혹은 시간 기반 평균값으로 결측을 대치.
    → 데이터의 연속성을 유지할 수 있어 시계열 분석에 적합.

4. 회귀 대치법 (Regression Imputation)
    → 결측이 있는 변수를 종속변수로 두고, 다른 변수로 회귀식을 만들어 예측값으로 대치.
    → 그러나 분산이 과소 추정되는 문제점이 있어, 확률적 회귀대치법으로 보완하기도 한다.

5. 다중 대치법 (Multiple Imputation, MICE)
    → 결측 대치를 여러 번 반복해 여러 개의 가상 데이터셋을 만들고, 이를 평균내어 최종 대치값을 얻는 방법.
        1. 대치단계 -> 2. 분석단계 -> 3. 결합단계


### 11.2. 이상치 처리

이상치(Outlier) -> 전체 데이터의 분포에서 극단적으로 벗어난 관측값

이상치 탐지 방법
---

1. EDA(탐색적 데이터 분석)와 박스플롯(Boxplot) 등을 통해 극단값 확인

2. 통계적으로는 IQR(사분위 범위) 또는 평균 ± k×표준편차 기준으로 이상치를 정의.

3. 평균은 이상치에 민감하기 때문에 중앙값(median)과 중위수 절대 편차(MAD)가 더 강건.

이상치 처리 방법
---

값 대체법 -> 하한값/상한값으로 대체 (Winsorizing).

가중치 조정법 -> 이상치의 영향력을 줄이는 방식.

단순 제거법 -> 분석 목적상 명백한 오류라면 삭제 가능.

**BUT 이상치가 무조건적인 제거 대상은 아님, 관점에 따라 달라질 수 있음을 인식**


### 11.3. 변수 구간화

변수 구간화 (Variable Binning)

변수 구간화(Binning) -> 연속형 변수를 범주형으로 변환하여 분석을 단순화하고, 모델 해석력을 높이는 과정

구간화의 목적과 방법
    -> 목적: 모델의 예측력 향상, 변수 해석의 용이성, 이상치 영향 완화
    -> 방법
        1. 클러스터링 기반 구간화 → 목표변수 없이 값의 유사성만으로 그룹화
        2. 의사결정나무 기반 구간화 → 타깃 변수와의 관계를 고려해 가장 적절한 구간으로 자동 분할

효과 평가 지표
    -> WOE (Weight of Evidence) → 구간별로 종속변수에 대한 설명력을 평가
    -> IV (Information Value) → 값이 높을수록 해당 변수가 종속변수를 잘 구분함을 의미.

### 11.4. 데이터 표준화와 정규화 스케일링

스케일링 → 서로 다른 단위를 가진 데이터를 일정한 기준으로 변환하여,변수 간 비교 가능성과 모델 학습 효율을 높이는 과정

표준화(Standardization)
    → 평균을 0, 표준편차를 1로 맞추는 변환 (Z-score 변환)
    → 각 값이 평균에서 얼마나 떨어져 있는지를 표현
    → 이상치에 다소 민감하지만, 변수 간 상대적 비교가 용이함

정규화(Normalization)
    → 데이터의 범위를 0~1 사이로 맞춤
    → 전체 데이터 중 상대적 위치를 파악할 때 유용
